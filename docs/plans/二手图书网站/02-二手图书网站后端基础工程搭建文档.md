# 二手图书网站后端基础工程搭建文档

**目标：** 从零创建一个独立的 `Spring Boot 3 + Java 17` 后端工程，作为校园二手图书网站的服务端基础骨架。同时把本项目第一阶段会用到的中间件统一通过 `Docker Compose` 启动，包括 `MySQL + Redis + RabbitMQ`，为后续用户、图书、订单、缓存、异步消息功能提前打好环境基础。

**架构说明：** 这一阶段只做“后端基础工程”，不急着把所有业务一次性写完。重点是把后端的工程目录、基础依赖、统一返回结构、全局异常处理、跨域配置、基础健康检查接口以及 Docker 中间件环境先稳定下来。这样你后面继续开发用户模块、图书模块、订单模块时，都是在同一套规范下往前推进。

**技术栈：** Java 17、Spring Boot 3.2.5、Spring Web、Spring Validation、MyBatis-Plus、MySQL 8、Redis 7、RabbitMQ 3 Management、Lombok、SpringDoc OpenAPI、Docker Compose、Maven

---

## 一、为什么第二阶段先做后端基础工程

真实项目开发里，很多人一上来就急着写“发布图书”“下单购买”这些业务接口，但如果基础工程没有先搭稳，后面通常会出现下面这些问题：

- 接口返回格式不统一
- 异常处理风格混乱
- 包结构越写越乱
- 环境配置混在一起，测试和开发不好切换
- 前后端联调时频繁碰到跨域问题
- 后面接 Redis、MQ 时又要反复改基础配置

所以这一步不是“可有可无”的准备动作，而是整个系统能不能长期稳定往下做的基础。

这一步的核心目标就是：

- 先把后端工程骨架搭起来
- 先把中间件环境统一用 Docker 跑起来
- 先把项目最基本的工程规范固定下来

---

## 二、这一阶段最终要完成什么

做完这一步后，你应该得到一个可以正常启动、可以正常连接数据库和中间件、可以对外提供基础接口的后端项目。

这一步包含：

- 创建独立后端工程 `second-hand-book-server`
- 创建 `docker-compose.yml` 启动 MySQL、Redis、RabbitMQ
- 配置 Spring Boot 基础依赖
- 配置 `application.yml` 和 `application-dev.yml`
- 建立统一包结构
- 建立统一返回对象
- 建立业务异常类和全局异常处理
- 建立跨域配置
- 建立 OpenAPI 文档配置
- 建立一个健康检查接口用于验证工程启动

这一步先不包含：

- 用户注册登录完整实现
- 图书表和订单表详细设计
- Redis 缓存业务逻辑
- RabbitMQ 异步消息业务逻辑
- 文件上传

这些会在后续阶段文档里逐步加上。

## 五、先启动中间件环境 `docker-compose.yml`

因为你已经明确要求中间件部署统一使用 Docker，所以从第二阶段开始，项目依赖环境就不再建议手工安装，而是统一走 `Docker Compose`。

### 1. 创建 `docker-compose.yml`

```yaml
services:
  mysql:
    image: mysql:8.0
    container_name: second-book-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: second_hand_book
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    command:
      - --default-authentication-plugin=mysql_native_password
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci
    volumes:
      - second-book-mysql-data:/var/lib/mysql

  redis:
    image: redis:7
    container_name: second-book-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    command:
      - redis-server
      - --appendonly
      - yes
    volumes:
      - second-book-redis-data:/data

  rabbitmq:
    image: rabbitmq:3-management
    container_name: second-book-rabbitmq
    restart: unless-stopped
    environment:
      RABBITMQ_DEFAULT_USER: guest
      RABBITMQ_DEFAULT_PASS: guest
      TZ: Asia/Shanghai
    ports:
      - "5672:5672"
      - "15672:15672"
    volumes:
      - second-book-rabbitmq-data:/var/lib/rabbitmq

volumes:
  second-book-mysql-data:
  second-book-redis-data:
  second-book-rabbitmq-data:
```

这段代码的作用：

- 使用 Docker 一次性启动当前阶段依赖的 3 个基础中间件
- MySQL 用于真实业务数据存储
- Redis 虽然当前阶段还没写业务缓存，但环境先准备好
- RabbitMQ 虽然当前阶段还没写异步消息，但环境先准备好
- 使用数据卷是为了避免容器重启后数据丢失

### 2. 启动命令

```powershell
docker compose up -d
```

这段代码的作用：

- 后台启动 MySQL、Redis、RabbitMQ 三个容器

### 3. 停止命令

```powershell
docker compose down
```

这段代码的作用：

- 停止当前项目使用的全部中间件容器

### 4. RabbitMQ 管理后台地址

```text
http://localhost:15672
账号: guest
密码: guest
```

这段代码的作用：

- 后面你接入 MQ 时，就可以从这里查看交换机、队列和消息情况

## 十二、创建统一返回对象 `Result.java`

```java
package com.example.booktrade.common.result;

import com.example.booktrade.common.enums.ResultCodeEnum;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Result<T> {

    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        return new Result<>(ResultCodeEnum.SUCCESS.getCode(), ResultCodeEnum.SUCCESS.getMessage(), data);
    }

    public static <T> Result<T> success(String message, T data) {
        return new Result<>(ResultCodeEnum.SUCCESS.getCode(), message, data);
    }

    public static <T> Result<T> fail(ResultCodeEnum resultCodeEnum) {
        return new Result<>(resultCodeEnum.getCode(), resultCodeEnum.getMessage(), null);
    }

    public static <T> Result<T> fail(Integer code, String message) {
        return new Result<>(code, message, null);
    }
}
```

这段代码的作用：

- 统一整个项目的接口返回格式
- 后面无论是成功、失败、校验报错还是业务异常，都走这个统一结构

## 十三、创建业务异常类 `BizException.java`

```java
package com.example.booktrade.common.exception;

import com.example.booktrade.common.enums.ResultCodeEnum;
import lombok.Getter;

@Getter
public class BizException extends RuntimeException {

    private final Integer code;

    public BizException(String message) {
        super(message);
        this.code = ResultCodeEnum.BAD_REQUEST.getCode();
    }

    public BizException(ResultCodeEnum resultCodeEnum) {
        super(resultCodeEnum.getMessage());
        this.code = resultCodeEnum.getCode();
    }

    public BizException(Integer code, String message) {
        super(message);
        this.code = code;
    }
}
```

这段代码的作用：

- 用于表达“这是业务可预期异常”，而不是系统崩溃异常
- 比如后面图书不存在、库存不足、订单状态错误，都适合抛这个异常

## 十四、创建全局异常处理器 `GlobalExceptionHandler.java`

```java
package com.example.booktrade.handler;

import com.example.booktrade.common.enums.ResultCodeEnum;
import com.example.booktrade.common.exception.BizException;
import com.example.booktrade.common.result.Result;
import jakarta.validation.ConstraintViolationException;
import org.springframework.validation.BindException;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BizException.class)
    public Result<Void> handleBizException(BizException ex) {
        return Result.fail(ex.getCode(), ex.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleMethodArgumentNotValidException(MethodArgumentNotValidException ex) {
        String message = ex.getBindingResult().getFieldError() == null
                ? ResultCodeEnum.BAD_REQUEST.getMessage()
                : ex.getBindingResult().getFieldError().getDefaultMessage();
        return Result.fail(ResultCodeEnum.BAD_REQUEST.getCode(), message);
    }

    @ExceptionHandler(BindException.class)
    public Result<Void> handleBindException(BindException ex) {
        String message = ex.getBindingResult().getFieldError() == null
                ? ResultCodeEnum.BAD_REQUEST.getMessage()
                : ex.getBindingResult().getFieldError().getDefaultMessage();
        return Result.fail(ResultCodeEnum.BAD_REQUEST.getCode(), message);
    }

    @ExceptionHandler(ConstraintViolationException.class)
    public Result<Void> handleConstraintViolationException(ConstraintViolationException ex) {
        return Result.fail(ResultCodeEnum.BAD_REQUEST.getCode(), ex.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception ex) {
        ex.printStackTrace();
        return Result.fail(ResultCodeEnum.SYSTEM_ERROR.getCode(), ResultCodeEnum.SYSTEM_ERROR.getMessage());
    }
}
```

这段代码的作用：

- 统一拦截全局异常
- 参数校验异常不会再直接抛一堆难看的错误给前端
- 业务异常和系统异常会以统一格式返回

## 十五、创建跨域配置 `CorsConfig.java`

```java
package com.example.booktrade.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.cors.CorsConfiguration;
import org.springframework.web.cors.UrlBasedCorsConfigurationSource;
import org.springframework.web.filter.CorsFilter;

import java.util.List;

@Configuration
public class CorsConfig {

    @Bean
    public CorsFilter corsFilter() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowCredentials(true);
        config.setAllowedOriginPatterns(List.of("*"));
        config.setAllowedHeaders(List.of("*"));
        config.setAllowedMethods(List.of("*"));
        config.setExposedHeaders(List.of("*"));

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        return new CorsFilter(source);
    }
}
```

这段代码的作用：

- 解决前后端分离开发时常见的跨域问题
- 后面 Vue3 本地开发时调用 Spring Boot 接口会更顺畅

## 二十二、常见问题排查

### 1. MySQL 连不上

常见原因：

- Docker 容器没启动
- 本机 `3306` 端口被占用
- `application-dev.yml` 里的密码和 Compose 不一致

解决方式：

- 先执行 `docker ps`
- 再检查 `docker-compose.yml`
- 再检查配置文件里的 MySQL 连接信息

### 2. Redis 连不上

常见原因：

- Redis 容器未启动
- 本机已有 Redis 占用了 `6379`

解决方式：

- 检查 `docker ps`
- 检查端口是否冲突

### 3. RabbitMQ 管理后台打不开

常见原因：

- 容器未启动
- `15672` 端口被占用

解决方式：

- 先检查容器状态
- 再确认浏览器访问的是 `http://localhost:15672`

### 4. Swagger 页面打不开

常见原因：

- 项目没启动成功
- `springdoc` 依赖没引入
- 访问地址写错

解决方式：

- 确认访问 `http://localhost:8080/swagger-ui.html`
- 查看控制台是否报错

### 5. 接口返回 500

常见原因：

- 配置类写错
- 包扫描路径不正确
- 依赖未下载完整

解决方式：

- 先执行 `mvn clean install`
- 再看启动日志中的报错位置
