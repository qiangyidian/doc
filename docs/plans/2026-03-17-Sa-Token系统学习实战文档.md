# Sa-Token系统学习实战文档

**目标：** 从零创建一个 Spring Boot 3 后台管理系统示例项目，系统学习 Sa-Token 在真实开发中的常见能力，包括登录、注销、登录校验、角色校验、权限校验、注解鉴权、路由拦截鉴权、会话信息读取、踢人下线、账号封禁、Redis 持久化。

**架构说明：** 这份文档使用 `Spring Boot + Sa-Token + MySQL + Redis + Docker Compose`。MySQL 负责存用户、角色、权限等基础数据；Redis 负责持久化 Sa-Token 的 token 与 session；业务上采用一个典型的后台管理系统场景，让你把登录认证、角色权限和后台管理动作串成一条完整链路来学习。

**技术栈：** Java 17、Spring Boot 3.2.5、Sa-Token Spring Boot3 Starter 1.45.0、Sa-Token Redis Template 1.45.0、MyBatis 3.0.3、MySQL 8.0、Redis 7、Docker Compose、Lombok

---

## 一、为什么学习 Sa-Token

Sa-Token 是一个专门解决“认证与鉴权”问题的 Java 框架。  
在真实开发里，你几乎绕不开下面这些问题：

1. 用户登录后，后端如何识别这是谁
2. 哪些接口必须登录后才能访问
3. 哪些接口只有管理员可以调用
4. 哪些用户有“新增用户”“删除订单”这类权限
5. 管理员如何把某个异常账号踢下线
6. 某个账号违规后，如何封禁一段时间
7. 项目重启后，登录状态如何不丢

Sa-Token 就是专门处理这些问题的。

根据 Sa-Token 官方首页，框架支持：

- 登录认证
- 权限认证
- 注解鉴权
- 路由鉴权
- 踢人下线
- 账号封禁
- Redis 集成

这也是这份文档会重点覆盖的内容。  
这里我使用的是 Sa-Token 当前官网首页显示的最新版本 `v1.45.0`，并且 Maven Central 上 `sa-token-spring-boot3-starter` 与 `sa-token-redis-template` 也都已经有 `1.45.0` 版本可用。

参考：

- [Sa-Token 官网首页](https://sa-token.cc/)
- [Sa-Token Spring Boot3 Starter 1.45.0](https://repo.maven.apache.org/maven2/cn/dev33/sa-token-spring-boot3-starter/)
- [Sa-Token Redis Template 1.45.0](https://repo.maven.apache.org/maven2/cn/dev33/sa-token-redis-template/)

## 二、这份文档里你会学到什么

这份文档不是只教你“怎么写一行 `StpUtil.login()`”，而是让你完成一个真实的后台权限示例项目。

你会学到：

1. 用户如何登录和注销
2. Sa-Token 如何判断当前用户是否已登录
3. 如何根据数据库里的角色和权限做鉴权
4. 如何使用 `@SaCheckLogin`、`@SaCheckRole`、`@SaCheckPermission`
5. 如何用路由拦截器统一拦截后台接口
6. 如何读取当前登录用户的 token 信息和 session 信息
7. 如何让管理员踢某个用户下线
8. 如何封禁某个用户一段时间
9. 如何把 Sa-Token 数据存进 Redis，而不是只存在内存里

## 四、先启动中间件 `docker-compose.yml`

```yaml
version: "3.9"

services:
  mysql:
    # 使用官方 MySQL 8.0 镜像。
    image: mysql:8.0
    container_name: sa-token-mysql
    environment:
      # root 密码仅用于本地练习。
      MYSQL_ROOT_PASSWORD: 123456
      # 第一次启动时自动创建这个数据库。
      MYSQL_DATABASE: sa_token_demo
    ports:
      - "3306:3306"
    command:
      # 改成 mysql_native_password，避免某些连接器在本地练习时报兼容问题。
      - --default-authentication-plugin=mysql_native_password
    volumes:
      # 把初始化 SQL 挂载进去。
      - ./init/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql

  redis:
    # 使用官方 Redis 镜像。
    image: redis:7
    container_name: sa-token-redis
    ports:
      - "6379:6379"
    command:
      # 开启 AOF，重启后数据更稳一些。
      - redis-server
      - --appendonly
      - yes
```

这段代码的作用：

- 启动 MySQL 和 Redis 两个最核心的中间件
- MySQL 保存用户、角色、权限数据
- Redis 保存 Sa-Token 的 token 和 session 数据
- Redis 打开 AOF，便于你理解“持久化 session”的意义

## 五、创建数据库脚本 `init/mysql/init.sql`

```sql
USE sa_token_demo;

-- 用户表。
CREATE TABLE IF NOT EXISTS sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    nickname VARCHAR(100) NOT NULL,
    status TINYINT NOT NULL DEFAULT 1,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 角色表。
CREATE TABLE IF NOT EXISTS sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code VARCHAR(50) NOT NULL UNIQUE,
    role_name VARCHAR(100) NOT NULL
);

-- 权限表。
CREATE TABLE IF NOT EXISTS sys_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    permission_code VARCHAR(100) NOT NULL UNIQUE,
    permission_name VARCHAR(100) NOT NULL
);

-- 用户角色关系表。
CREATE TABLE IF NOT EXISTS sys_user_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    UNIQUE KEY uk_user_role (user_id, role_id)
);

-- 角色权限关系表。
CREATE TABLE IF NOT EXISTS sys_role_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id BIGINT NOT NULL,
    permission_id BIGINT NOT NULL,
    UNIQUE KEY uk_role_permission (role_id, permission_id)
);

-- 初始化用户：admin 和普通用户。
INSERT INTO sys_user (id, username, password, nickname, status)
VALUES
    (1, 'admin', '123456', '超级管理员', 1),
    (2, 'test', '123456', '普通测试用户', 1)
ON DUPLICATE KEY UPDATE nickname = VALUES(nickname);

-- 初始化角色。
INSERT INTO sys_role (id, role_code, role_name)
VALUES
    (1, 'admin', '管理员'),
    (2, 'user', '普通用户')
ON DUPLICATE KEY UPDATE role_name = VALUES(role_name);

-- 初始化权限。
INSERT INTO sys_permission (id, permission_code, permission_name)
VALUES
    (1, 'user.query', '查看用户'),
    (2, 'user.add', '新增用户'),
    (3, 'user.kickout', '踢人下线'),
    (4, 'user.disable', '封禁用户')
ON DUPLICATE KEY UPDATE permission_name = VALUES(permission_name);

-- 用户与角色关系：admin 拥有 admin 角色，test 拥有 user 角色。
INSERT INTO sys_user_role (user_id, role_id)
VALUES
    (1, 1),
    (2, 2)
ON DUPLICATE KEY UPDATE role_id = VALUES(role_id);

-- 角色与权限关系：管理员拥有全部权限，普通用户只有查看权限。
INSERT INTO sys_role_permission (role_id, permission_id)
VALUES
    (1, 1),
    (1, 2),
    (1, 3),
    (1, 4),
    (2, 1)
ON DUPLICATE KEY UPDATE permission_id = VALUES(permission_id);
```

这段代码的作用：

- 创建 Sa-Token 学习所需的最小 RBAC 表结构
- 提前插入一个管理员和一个普通用户，方便你直接测试
- 角色和权限关系已经准备好，后面登录后就可以直接验证

  # token 名称，前端请求时默认用这个名字传 token。
  token-name: satoken
  # token 有效期，单位秒。这里是 7 天。
  timeout: 604800
  # token 最后活跃时间，超过 30 分钟不操作就会被视为失活。
  active-timeout: 1800
  # 是否允许同一账号多地同时登录。
  is-concurrent: true
  # 在多人登录同一账号时，是否共享同一个 token。
  # false 表示每次登录都会生成新的 token，更适合后台系统演示。
  is-share: false
  # token 风格。
  token-style: uuid
  # 是否打印 Sa-Token 的日志。
  is-log: true

这段代码的作用：

### 9.2 创建 `common/GlobalExceptionHandler.java`

```java
package com.example.satokenadmin.common;

import cn.dev33.satoken.exception.DisableServiceException;
import cn.dev33.satoken.exception.NotLoginException;
import cn.dev33.satoken.exception.NotPermissionException;
import cn.dev33.satoken.exception.NotRoleException;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(NotLoginException.class)
    public Result<String> handleNotLogin(NotLoginException e) {
        return Result.fail("当前未登录，请先登录后再访问");
    }

    @ExceptionHandler(NotPermissionException.class)
    public Result<String> handleNotPermission(NotPermissionException e) {
        return Result.fail("没有权限，缺少权限码：" + e.getPermission());
    }

    @ExceptionHandler(NotRoleException.class)
    public Result<String> handleNotRole(NotRoleException e) {
        return Result.fail("没有角色，缺少角色标识：" + e.getRole());
    }

    @ExceptionHandler(DisableServiceException.class)
    public Result<String> handleDisable(DisableServiceException e) {
        return Result.fail("账号已被封禁，剩余封禁时间（秒）：" + e.getDisableTime());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<String> handleValid(MethodArgumentNotValidException e) {
        return Result.fail("参数校验失败：" + e.getBindingResult().getFieldError().getDefaultMessage());
    }

    @ExceptionHandler(IllegalArgumentException.class)
    public Result<String> handleIllegalArgument(IllegalArgumentException e) {
        return Result.fail(e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public Result<String> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.fail("系统异常：" + e.getMessage());
    }
}
```

这段代码的作用：

- 把 Sa-Token 抛出的常见异常转换成更友好的中文响应
- 例如未登录、无权限、无角色、账号被封禁时，都能直接看到明确提示

    }
}

## 十七、创建 Service 实现类

### 17.1 创建 `service/impl/AuthServiceImpl.java`

```java
package com.example.satokenadmin.service.impl;

import cn.dev33.satoken.session.SaSession;
import cn.dev33.satoken.stp.StpUtil;
import com.example.satokenadmin.dto.LoginRequest;
import com.example.satokenadmin.dto.LoginResponse;
import com.example.satokenadmin.entity.SysUser;
import com.example.satokenadmin.mapper.SysPermissionMapper;
import com.example.satokenadmin.mapper.SysRoleMapper;
import com.example.satokenadmin.mapper.SysUserMapper;
import com.example.satokenadmin.service.AuthService;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

import java.util.HashMap;
import java.util.List;
import java.util.Map;
import java.util.Objects;

@Service
@RequiredArgsConstructor
public class AuthServiceImpl implements AuthService {

    private final SysUserMapper sysUserMapper;
    private final SysRoleMapper sysRoleMapper;
    private final SysPermissionMapper sysPermissionMapper;

    @Override
    public LoginResponse login(LoginRequest request) {
        SysUser user = sysUserMapper.selectByUsername(request.getUsername());
        if (user == null) {
            throw new IllegalArgumentException("用户名不存在");
        }
        if (!Objects.equals(user.getPassword(), request.getPassword())) {
            throw new IllegalArgumentException("密码错误");
        }
        if (!Objects.equals(user.getStatus(), 1)) {
            throw new IllegalArgumentException("当前账号已被业务停用");
        }

        // 如果这个账号此前被 Sa-Token 封禁了，checkDisable 会直接抛异常。
        StpUtil.checkDisable(user.getId());

        // 执行登录。
        // 只需要传入登录用户的唯一标识即可，这里传用户 ID。
        StpUtil.login(user.getId());

        // 登录成功后，可以往当前会话里放一些业务数据。
        // 这些数据会跟随当前登录态一起保存到 Redis。
        SaSession session = StpUtil.getSession();
        session.set("userId", user.getId());
        session.set("username", user.getUsername());
        session.set("nickname", user.getNickname());

        List<String> roleList = sysRoleMapper.selectRoleCodeListByUserId(user.getId());
        List<String> permissionList = sysPermissionMapper.selectPermissionCodeListByUserId(user.getId());

        LoginResponse response = new LoginResponse();
        response.setUserId(user.getId());
        response.setUsername(user.getUsername());
        response.setNickname(user.getNickname());
        response.setTokenName(StpUtil.getTokenInfo().getTokenName());
        response.setTokenValue(StpUtil.getTokenInfo().getTokenValue());
        response.setTokenTimeout(StpUtil.getTokenInfo().getTokenTimeout());
        response.setRoleList(roleList);
        response.setPermissionList(permissionList);

        Map<String, Object> sessionData = new HashMap<>();
        sessionData.put("userId", session.get("userId"));
        sessionData.put("username", session.get("username"));
        sessionData.put("nickname", session.get("nickname"));
        response.setSessionData(sessionData);
        return response;
    }

    @Override
    public void logout() {
        // 退出登录后，当前 token 对应的登录态会被清除。
        StpUtil.logout();
    }

    @Override
    public Map<String, Object> currentLoginInfo() {
        // 先检查当前请求是否已登录。
        StpUtil.checkLogin();

        Map<String, Object> data = new HashMap<>();
        data.put("loginId", StpUtil.getLoginIdAsLong());
        data.put("tokenName", StpUtil.getTokenInfo().getTokenName());
        data.put("tokenValue", StpUtil.getTokenInfo().getTokenValue());
        data.put("tokenTimeout", StpUtil.getTokenInfo().getTokenTimeout());
        data.put("isLogin", StpUtil.isLogin());
        data.put("sessionUsername", StpUtil.getSession().get("username"));
        data.put("sessionNickname", StpUtil.getSession().get("nickname"));
        return data;
    }
}
```

这段代码的作用：

- 完整演示了 Sa-Token 最核心的登录和注销流程
- 登录成功后，把用户信息放进 session，便于你理解 Sa-Token 的会话能力
- 同时把 token 信息、角色、权限一起返回，方便你测试

## 二十、你应该怎么一步一步测试

下面我给你一套完整的学习顺序。  
你不要一上来就测“踢人下线”或者“封禁”，先按顺序来。

### 第 1 步：管理员登录

```bash
curl.exe -X POST "http://localhost:8080/auth/login" -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"123456\"}"
```

你应该看到：

- 返回 `tokenName`
- 返回 `tokenValue`
- 返回当前用户角色列表和权限列表

重点观察：

- 管理员会拥有 `admin` 角色
- 管理员会拥有 `user.query`、`user.add`、`user.kickout`、`user.disable` 这些权限

### 第 2 步：把 token 带上，再查当前登录信息

假设你上一步返回的 token 是：

```text
satoken=xxxxxx
```

那你后面调用接口时，要在请求头里带上：

```text
satoken: xxxxxx
```

测试接口：

```bash
curl.exe "http://localhost:8080/auth/login-info" -H "satoken: 这里换成你的token"
```

你应该看到：

- 当前登录用户 ID
- token 值
- token 剩余有效期
- session 中保存的 username、nickname

### 第 3 步：查看个人信息

```bash
curl.exe "http://localhost:8080/user/profile" -H "satoken: 这里换成你的token"
```

你应该看到：

- 当前登录用户的完整信息

这个接口重点学习：

- `@SaCheckLogin`
- `StpUtil.getLoginIdAsLong()`

### 第 3.5 步：去 Redis 看 Sa-Token 持久化的数据

```bash
docker exec -it sa-token-redis redis-cli KEYS "satoken:*"
```

你应该看到：

- Redis 里已经出现了一批以 `satoken:` 开头的 key

这个步骤的意义：

- 让你亲眼看到“Sa-Token + Redis”不是一句概念
- 登录成功后，token、session、登录状态确实已经被写到了 Redis

### 第 4 步：管理员查询用户列表

```bash
curl.exe "http://localhost:8080/admin/users" -H "satoken: 这里换成管理员token"
```

你应该看到：

- 用户列表成功返回

这个接口重点学习：

- `@SaCheckPermission("user.query")`

### 第 5 步：管理员新增用户

```bash
curl.exe -X POST "http://localhost:8080/admin/users" -H "Content-Type: application/json" -H "satoken: 这里换成管理员token" -d "{\"username\":\"newuser\",\"password\":\"123456\",\"nickname\":\"新用户\"}"
```

你应该看到：

- 新用户创建成功

这个接口重点学习：

- `@SaCheckPermission("user.add")`

### 第 6 步：普通用户登录，再访问管理员接口

普通用户登录：

```bash
curl.exe -X POST "http://localhost:8080/auth/login" -H "Content-Type: application/json" -d "{\"username\":\"test\",\"password\":\"123456\"}"
```

然后让普通用户访问管理员接口：

```bash

```

你应该看到：

- 接口被拒绝
- 提示缺少权限 `user.query` 或缺少管理员角色

- 让你直观看到“角色和权限不是摆设，确实会影响接口访问结果”

### 第 7 步：管理员踢普通用户下线

先用管理员 token 调用：

curl.exe -X POST "http://localhost:8080/admin/kickout/2" -H "satoken: 这里换成管理员token"

- 第二次请求失败

- `StpUtil.kickout(userId)`

这个步骤重点学习：

- `StpUtil.disable(userId, seconds)`
- `StpUtil.checkDisable(userId)`

- `isDisable`
- `disableSeconds`

- 请求失败
- 提示未登录

## 二十一、这一套项目里 Sa-Token 的能力是怎么体现的

### 1. 登录认证

对应代码：

- `AuthServiceImpl.login()`
- `AuthController.login()`

核心 API：

- `StpUtil.login(userId)`

对应代码：

- `StpInterfaceImpl.getRoleList()`

核心 API：

- `StpUtil.checkRole("admin")`

### 4. 权限校验

对应代码：

- `StpInterfaceImpl.getPermissionList()`

核心 API：

- `StpUtil.checkPermission("user.add")`

### 5. Session 会话

对应代码：

- `AuthServiceImpl.login()`
- `AuthServiceImpl.currentLoginInfo()`

核心 API：

- `StpUtil.getSession()`

### 6. 踢人下线

对应代码：

- `UserServiceImpl.kickout()`

核心 API：

- `StpUtil.kickout(userId)`

### 7. 账号封禁

对应代码：

- `UserServiceImpl.disable()`
- `AuthServiceImpl.login()`

核心 API：

- `StpUtil.disable(userId, seconds)`
- `StpUtil.checkDisable(userId)`

### 8. Redis 持久化

对应位置：

- `pom.xml` 中的 `sa-token-redis-template`

- 没接 Redis 时，Sa-Token 数据默认保存在内存
- 接了 Redis 后，token 与 session 会持久化到 Redis，项目重启后登录状态更稳

## 二十二、常见报错排查

### 1. 登录成功了，但后续接口一直提示未登录

常见原因：

- 你没有在请求头里带 `satoken`
- token 名和请求头名写错了
- 你复制 token 时带了多余空格

解决方式：

- 看登录接口返回的 `tokenName`
- 后续请求头要写成：

```text
satoken: 你的token值
```

### 2. 报“没有权限”或“没有角色”

常见原因：

- 当前用户在数据库里没有绑定对应角色
- 当前角色没有绑定对应权限

解决方式：

- 检查 `sys_user_role`
- 检查 `sys_role_permission`

### 3. 项目重启后 token 丢了

常见原因：

- 没接 Redis 持久化插件
- Redis 没启动

解决方式：

- 检查 `pom.xml` 里是否有 `sa-token-redis-template`
- 检查 Redis 是否正常启动

### 4. 新增用户后登录失败

常见原因：

- 你只是把用户插入了 `sys_user`
- 但没有给这个用户绑定角色

说明：

- 这份示例里“新增用户”只是演示 `user.add` 权限本身
- 如果你想让新用户也拥有角色和权限，需要继续往 `sys_user_role` 里插数据

### 5. 封禁了用户，但还是能继续访问旧接口

说明：

- 账号封禁主要影响“后续登录”和相关封禁校验逻辑
- 如果你想更严格，可以在关键接口额外调用 `StpUtil.checkDisable(StpUtil.getLoginId())`

## 二十三、这份文档最适合怎么学

我建议你按下面顺序来练：

1. 先把 Docker 中间件启动起来
