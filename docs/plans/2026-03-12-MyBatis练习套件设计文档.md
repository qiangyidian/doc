# MyBatis 练习套件设计文档

**Date:** 2026-03-12

## 目标

提供 3 个完全独立的 Spring Boot + MyBatis 练习项目：

1. `mybatis-user-crud-demo`：练习单表 CRUD
2. `mybatis-order-detail-demo`：练习主从表一对多详情查询
3. `mybatis-product-search-demo`：练习动态 SQL 条件检索和分页

目标读者是 MyBatis 初学者，需要中文 Markdown 文档，并且可以照着文档一步一步创建项目、创建文件、复制代码、启动 Docker 和测试接口。

## 为什么选择这 3 个场景

这 3 个场景基本覆盖了 MyBatis 在生产中最常见的 3 类使用方式：

- 单表增删改查
- 多表关联查询
- 动态 SQL 列表检索

学完这 3 个项目后，初学者基本就能理解：

- MyBatis 为什么适合“自己掌控 SQL”
- `Mapper 接口 + XML` 是怎么协作的
- `resultMap`、`if`、`where`、`limit` 这些标签和写法分别解决什么问题

## 为什么仍然拆成 3 个独立项目

沿用你之前确认过的 MQ 和 Redis 交付方式：

- 每个场景单独一个 Spring Boot 项目
- 每个项目单独一份 `docker-compose.yml`
- 每个项目单独一个数据库
- 每个项目都可以独立启动、独立练习、独立删除

这样最适合小白反复练习，不会把多个知识点混在一个工程里。

## 3 个生产风格场景

### 1. 用户管理单表 CRUD

生产意义：

- 后台管理系统最常见的就是用户、角色、菜单、字典这类单表 CRUD
- MyBatis 很适合这种“SQL 简单但我想自己掌控”的场景

练习目标：

- 新增用户
- 根据 ID 查询用户
- 查询用户列表
- 更新用户
- 删除用户

### 2. 订单详情一对多查询

生产意义：

- 订单主表和订单项表是经典的一对多结构
- 业务里经常需要“查一个订单，同时把它的明细项一起查出来”
- 这是 `resultMap + collection` 的典型练习场景

练习目标：

- 查询订单详情
- 同时带出订单项列表
- 理解一对多映射关系

### 3. 商品后台动态条件搜索

生产意义：

- 管理后台最常见的是列表页搜索
- 条件可能很多，而且不是每个条件都会传
- 这是 MyBatis 动态 SQL 的经典使用场景

练习目标：

- 关键字查询
- 分类筛选
- 状态筛选
- 价格区间筛选
- 分页查询

  - 应用端口：`8102`
  - MySQL：`3314`

  - 应用端口：`8103`

## 推荐学习顺序

1. `mybatis-user-crud-demo`
2. `mybatis-order-detail-demo`
3. `mybatis-product-search-demo`

原因：

- 先掌握最基础的 `Mapper + XML + CRUD`
- 再学一对多映射
- 最后学动态 SQL 和分页
