# Spring Boot + Spring AI + PGVector RAG 精排与上下文压缩实现文档

**目标：** 基于上一份 `2026-04-14-RAG检索增强实现文档.md` 继续升级项目，完整演示“混合检索候选召回 -> Reranker 精排 -> 上下文去重与压缩 -> Prompt 拼装 -> 大模型回答”这条第三阶段在线问答增强链路。

**架构说明：** 这份文档严格承接第二阶段的检索增强版 RAG 项目，不改变已经完成的 `目录同步`、`查询改写`、`向量召回 + 关键词召回`、`RRF 融合` 和 `召回评估`，而是在在线问答链路中进一步加入 `Reranker 精排` 和 `上下文压缩与裁剪`。这一阶段的重点，不再是“把更多片段拉回来”，而是“把最值得给模型看的片段排在前面，并且把噪声压掉”。

**技术栈：** Java 17、Spring Boot 3.5.x、Spring AI 1.1.x、Spring Web、MyBatis、PostgreSQL 16、PGVector、`pg_trgm`、Docker Compose

---

## 一、这一阶段到底解决什么问题

如果你已经把第二阶段做完，你会得到一个“已经比较像回事”的 RAG 系统：

1. 会目录同步
2. 会增量更新
3. 会查询改写
4. 会混合召回
5. 会做基础召回评估

但这时候新的问题也会马上冒出来：

1. 相关片段虽然召回到了，但排序还是不够稳定
2. 多路召回后经常会出现“主题相近、内容重复”的片段
3. 候选片段越多，Prompt 里的噪声越大
4. 模型真正生成答案时，最重要的信息可能被埋在中间
5. Token 成本也会随着候选上下文不断上涨

所以，第三阶段的核心目标非常明确：

1. 让“召回结果”进入一次**精排**
2. 让“准备喂给模型的上下文”进入一次**压缩**

换句话说，第二阶段解决的是“有没有找到”，第三阶段解决的是“找到以后，怎么更好地交给模型”。

## 二、先讲清楚这一阶段的取舍

第三阶段最容易上头的地方，是一上来就想接很多新模型和新中间件。

但教程式项目不适合这样做。

为了保持与前两阶段的衔接平滑，我这里明确做 3 个取舍。

### 1、Reranker 先不接独立 Cross-Encoder 服务，先用 LLM 结构化精排

这里我先把 3 种方案摆清楚：

#### 方案 A：接独立 Reranker 模型服务

优点：

- 最像真实生产方案
- 专门做相关性打分，通常效果更稳

缺点：

- 需要新增模型部署或额外供应商
- 教程阶段会把复杂度一下拉高

#### 方案 B：继续沿用当前 `ChatClient`，做结构化 LLM 精排

优点：

- 不需要引入新基础设施
- 和现有 `Spring AI` 技术栈完全兼容
- 很适合教程式实现文档，能把精排逻辑讲得非常透明

缺点：

- 成本和延迟通常高于专用 Reranker
- 一致性不如独立 Cross-Encoder 稳定

#### 方案 C：只靠规则排序，不做模型精排

优点：

- 实现最简单

缺点：

- 对复杂语义相关性帮助有限
- 只能当过渡方案，不适合作为本阶段主线

**这一份文档我推荐并采用方案 B。**

原因很直接：

1. 你当前项目已经有 `ChatClient`
2. Spring AI 官方文档明确支持结构化输出转换
3. 这最适合作为教程阶段的第三步落地

### 2、上下文压缩采用“两段式”，不一上来就做复杂长链路

这一阶段的上下文压缩，我不建议你一步到位做成特别复杂的流水线。

这份文档采用的方案是：

1. **规则压缩**：去重、丢弃低价值片段、限制每片段长度
2. **LLM 压缩**：针对每个高价值片段，抽取与问题最相关的信息

为什么这样设计？

因为只做规则压缩，很多“看起来长、但其实关键句就在中间”的片段压不准；
而只做 LLM 压缩，又会让成本和复杂度一下变高。

两段式是一个非常适合教学落地的折中方案。

### 3、这一阶段不改数据库结构

这一点很重要。

第三阶段的变化，全部发生在**在线问答链路**里：

1. 召回后的候选如何重排
2. 重排后的候选如何压缩
3. 压缩后的上下文如何喂给模型

所以这一阶段：

1. 不需要新增数据表
2. 不需要修改 `init.sql`
3. 不需要重建 PostgreSQL 数据卷

这也是为什么第三阶段的开发节奏会比第二阶段轻一些。

## 三、这一阶段的整体架构

第三阶段其实是在第二阶段在线问答管道中，再插入两个关键模块。

### 1、第二阶段在线问答管道

```text
用户问题
    ->
查询改写
    ->
混合检索
    ->
融合排序
    ->
直接拼 Prompt
    ->
模型回答
```

### 2、第三阶段在线问答管道

```text
用户问题
    ->
查询改写
    ->
混合检索
    ->
候选片段
    ->
Reranker 精排
    ->
TopN 高质量片段
    ->
上下文压缩与裁剪
    ->
更短、更聚焦的上下文
    ->
Prompt 拼装
    ->
模型回答
```

### 3、第三阶段的两个新增模块分别做什么

#### Reranker 精排

职责：

1. 对第二阶段召回回来的候选片段重新打分
2. 把和问题更相关的片段排到前面
3. 淘汰明显相关性不够的候选

#### 上下文压缩

职责：

1. 去掉重复片段
2. 去掉与问题无关的描述
3. 保留更短、更有信息密度的内容
4. 降低最终 Prompt 噪声和成本

## 五、先说明两件“不需要动”的事

### 5.1 `pom.xml` 本阶段通常不需要新增依赖

为什么？

因为这一阶段我们继续沿用：

1. `spring-ai-starter-model-openai`
2. `spring-ai-starter-vector-store-pgvector`
3. `ChatClient`
4. Spring AI 结构化输出转换能力

这些在前两阶段已经具备。

所以如果你完全按前两份文档做下来，第三阶段通常不需要再额外加依赖。

### 5.2 `init.sql` 本阶段不需要再改

为什么？

因为 Reranker 和上下文压缩都发生在查询链路里，不依赖新的表结构。

这意味着：

1. 这一阶段不用再 `docker compose down -v`
2. 你不需要重建 PostgreSQL
3. 重点都在 Java 服务层逻辑

## 六、修改 `src/main/resources/application.yml`

第三阶段要新增两类配置：

1. 精排配置
2. 压缩配置

同时，我们还要对第二阶段里“融合后保留多少条”这个概念做一个升级：

- 第二阶段的融合结果，现在不再直接作为最终上下文
- 它会变成“Reranker 的候选集”

请把 `application.yml` 升级为下面这个版本：

```yaml

    username: postgres
    password: postgres
    driver-class-name: org.postgresql.Driver

  ai:
    openai:
      api-key: ${OPENAI_API_KEY:please-set-your-openai-api-key}
      base-url: ${OPENAI_BASE_URL:https://api.openai.com}

      chat:
        options:
          model: gpt-4o-mini
          temperature: 0.2

      embedding:
        options:
          model: text-embedding-3-small

    vectorstore:
      pgvector:
        initialize-schema: false
        dimensions: 1536
        table-name: vector_store

mybatis:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true

rag:
  knowledge-base:
    default-category: tutorial
    chunk-size: 500
    chunk-overlap: 80
    default-directory: C:/Gitee/RAG/knowledge-base/tutorial
    recursive-sync: true

  retrieval:
    vector-top-k: 6
    similarity-threshold: 0.45
    keyword-top-k: 6
    trgm-similarity-threshold: 0.08

    # 这里不再表示“最终给模型的片段数”，
    # 而是表示“进入 Reranker 的候选片段数”。
    candidate-top-k: 8

    rrf-k: 60
    query-rewrite-enabled: true
    rewrite-query-count: 2

  rerank:
    enabled: true
    # 精排后最终保留多少片段进入压缩阶段。
    top-k: 4
    # 如果得分过低，就直接淘汰。
    min-score: 55
    # 精排时单条候选展示给模型的最大字符数。
    max-content-length-per-chunk: 350

  compression:
    enabled: true
    # 是否先做规则去重。
    deduplicate-before-llm: true
    # 单片段压缩后允许的最大字符数。
    max-chars-per-chunk: 260

    drop-empty-chunks: true

    case-file: retrieval-eval/tutorial-eval-cases.json
```

这段代码的作用：

- 把第三阶段新增的精排与压缩参数显式配置化
- 让“候选召回数量”和“最终进入答案上下文的数量”正式分开

## 八、修改 `src/main/java/com/qiange/ragdemo/config/RagProperties.java`

第三阶段最重要的配置升级，就在这里。

```java
package com.qiange.ragdemo.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;

/**
 * 第三阶段 RAG 配置绑定类。
 *
 * 和第二阶段相比，核心新增了两组配置：
 * 1. rerank：控制精排
 * 2. compression：控制上下文压缩
 */
@Data
@ConfigurationProperties(prefix = "rag")
public class RagProperties {

    private KnowledgeBase knowledgeBase = new KnowledgeBase();

    private Retrieval retrieval = new Retrieval();

    private Rerank rerank = new Rerank();

    private Compression compression = new Compression();

    private Evaluation evaluation = new Evaluation();

    @Data
    public static class KnowledgeBase {

        private String defaultCategory = "tutorial";

        private Integer chunkSize = 500;

        private Integer chunkOverlap = 80;

        private String defaultDirectory;

        private Boolean recursiveSync = true;
    }

    @Data
    public static class Retrieval {

        private Integer vectorTopK = 6;

        private Double similarityThreshold = 0.45;

        private Integer keywordTopK = 6;

        private Double trgmSimilarityThreshold = 0.08;

        /**
         * 混合检索后，送给 Reranker 的候选数量。
         */
        private Integer candidateTopK = 8;

        private Integer rrfK = 60;

        private Boolean queryRewriteEnabled = true;

        private Integer rewriteQueryCount = 2;
    }

    @Data
    public static class Rerank {

        /**
         * 是否开启精排。
         */
        private Boolean enabled = true;

        /**
         * 精排后最终保留的片段数量。
         */
        private Integer topK = 4;

        /**
         * 精排最低得分阈值。
         * 分数低于该值的片段会被丢弃。
         */
        private Integer minScore = 55;

        /**
         * 精排时每个候选片段传给模型的最大字符数。
         */
        private Integer maxContentLengthPerChunk = 350;
    }

    @Data
    public static class Compression {

        /**
         * 是否开启上下文压缩。
         */
        private Boolean enabled = true;

        /**
         * 是否在进入 LLM 压缩前先做规则去重。
         */
        private Boolean deduplicateBeforeLlm = true;

        /**
         * 每个片段压缩后的最大字符数。
         */
        private Integer maxCharsPerChunk = 260;

        /**
         * 最终整段上下文允许的最大字符数。
         */
        private Integer maxTotalContextChars = 1800;

        /**
         * 压缩后为空的片段是否直接丢弃。
         */
        private Boolean dropEmptyChunks = true;
    }

    @Data
    public static class Evaluation {

        private String caseFile;
    }
}
```

这段代码的作用：

- 把第三阶段的精排和压缩参数都变成强类型配置
- 为后面的 `RerankService` 和 `ContextCompressionService` 提供统一参数来源

## 九、修改和新增 DTO

第三阶段开始，问答返回结果不能再只停留在“答案 + 引用片段”。

因为现在我们有了新的在线问答流水线阶段：

1. 候选召回
2. 精排
3. 压缩
4. 最终答案

所以，返回结果里必须开始体现这些阶段信息。

### 9.1 修改 `src/main/java/com/qiange/ragdemo/dto/ReferenceChunkResponse.java`

```java
import lombok.Builder;
```

这段代码的作用：

- 让单个引用片段可以完整反映“召回 -> 精排 -> 压缩 -> 最终是否入选”的全过程
- 对调试和教学都非常有价值

### 12.1 修改 `src/main/java/com/qiange/ragdemo/service/impl/HybridRetrievalServiceImpl.java`

第二阶段里，这个类的职责是“把融合后的结果直接作为最终结果返回”。

到了第三阶段，它的职责要调整成：

- 返回“给 Reranker 用的候选集”

也就是说，它仍然负责召回和融合，但不再负责决定“最后哪些片段进入答案上下文”。

```java
package com.qiange.ragdemo.service.impl;

import com.qiange.ragdemo.common.RagConstants;
import com.qiange.ragdemo.config.RagProperties;
import com.qiange.ragdemo.repository.KeywordSearchRepository;
import com.qiange.ragdemo.service.HybridRetrievalService;
import com.qiange.ragdemo.service.model.RetrievedChunk;
import lombok.RequiredArgsConstructor;
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.stereotype.Service;
import org.springframework.util.DigestUtils;
import org.springframework.util.StringUtils;

import java.nio.charset.StandardCharsets;
import java.util.ArrayList;
import java.util.Comparator;
import java.util.LinkedHashMap;
import java.util.List;
import java.util.Map;

/**
 * 第三阶段混合检索服务实现。
 *
 * 注意这里的输出语义已经变化：
 * 返回的是“候选片段”，而不是最终上下文。
 */
@Service
@RequiredArgsConstructor
public class HybridRetrievalServiceImpl implements HybridRetrievalService {

    private final VectorStore vectorStore;

    private final KeywordSearchRepository keywordSearchRepository;

    private final RagProperties ragProperties;

    @Override
    public List<RetrievedChunk> retrieve(List<String> queries, String category) {
        Map<String, RetrievedChunk> mergedChunkMap = new LinkedHashMap<>();

        for (String query : queries) {
            List<Document> vectorDocuments = vectorSearch(query, category);
            mergeByRrf(mergedChunkMap, vectorDocuments, RagConstants.RETRIEVAL_SOURCE_VECTOR);

            List<Document> keywordDocuments = keywordSearchRepository.search(
                    query,
                    category,
                    ragProperties.getRetrieval().getKeywordTopK(),
                    ragProperties.getRetrieval().getTrgmSimilarityThreshold()
            );
            mergeByRrf(mergedChunkMap, keywordDocuments, RagConstants.RETRIEVAL_SOURCE_KEYWORD);
        }

        return mergedChunkMap.values().stream()
                .sorted(Comparator.comparing(RetrievedChunk::getFusionScore).reversed())
                .limit(ragProperties.getRetrieval().getCandidateTopK())
                .toList();
    }

    private List<Document> vectorSearch(String query, String category) {
        SearchRequest.Builder builder = SearchRequest.builder()
                .query(query)
                .topK(ragProperties.getRetrieval().getVectorTopK())
                .similarityThreshold(ragProperties.getRetrieval().getSimilarityThreshold());

        if (StringUtils.hasText(category)) {
            builder.filterExpression("category == '" + escapeFilterValue(category) + "'");
        }

        List<Document> documents = vectorStore.similaritySearch(builder.build());
        return documents == null ? new ArrayList<>() : documents;
    }

    private void mergeByRrf(Map<String, RetrievedChunk> mergedChunkMap,
                            List<Document> documents,
                            String source) {
        for (int i = 0; i < documents.size(); i++) {
            Document document = documents.get(i);
            String uniqueKey = buildUniqueKey(document);
            double deltaScore = 1.0 / (ragProperties.getRetrieval().getRrfK() + i + 1);

            RetrievedChunk existingChunk = mergedChunkMap.get(uniqueKey);
            if (existingChunk == null) {
                mergedChunkMap.put(uniqueKey, RetrievedChunk.builder()
                        .uniqueKey(uniqueKey)
                        .document(document)
                        .retrievalSource(source)
                        .fusionScore(deltaScore)
                        .rerankScore(null)
                        .compressedContent(null)
                        .selectedForAnswer(false)
                        .build());
                continue;
            }

            existingChunk.setFusionScore(existingChunk.getFusionScore() + deltaScore);
            existingChunk.setRetrievalSource(mergeRetrievalSource(
                    existingChunk.getRetrievalSource(),
                    source
            ));
        }
    }

    private String buildUniqueKey(Document document) {
        Object sourcePath = document.getMetadata().get(RagConstants.METADATA_SOURCE_PATH);
        Object chunkIndex = document.getMetadata().get(RagConstants.METADATA_CHUNK_INDEX);

        if (sourcePath != null && chunkIndex != null) {
            return sourcePath + "#" + chunkIndex;
        }

        return DigestUtils.md5DigestAsHex(document.getText().getBytes(StandardCharsets.UTF_8));
    }

    private String mergeRetrievalSource(String current, String incoming) {
        if (current == null || current.equals(incoming)) {
            return incoming;
        }
        return RagConstants.RETRIEVAL_SOURCE_HYBRID;
    }

    private String escapeFilterValue(String value) {
        return value.replace("'", "\\'");
    }
}
```

- 保留第二阶段的混合召回与 RRF 融合能力

### 12.2 创建 `src/main/java/com/qiange/ragdemo/service/impl/RerankServiceImpl.java`

这一版精排服务采用 `ChatClient + BeanOutputConverter` 来实现结构化 LLM 精排。

```java
package com.qiange.ragdemo.service.impl;

import com.qiange.ragdemo.config.RagProperties;
import com.qiange.ragdemo.service.RerankService;
import com.qiange.ragdemo.service.model.RerankDecision;
import com.qiange.ragdemo.service.model.RerankResultEnvelope;
import com.qiange.ragdemo.service.model.RetrievedChunk;
import lombok.RequiredArgsConstructor;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.converter.BeanOutputConverter;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.Comparator;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

/**
 * 第三阶段精排服务实现。
 *
 * 这里采用的策略是：
 * 1. 先把候选片段整理成一个稳定、可控的列表
 * 2. 再让模型按照结构化格式返回每条片段的相关性评分
 * 3. 最后按分数排序，并应用 topK / minScore 过滤
 */
@Service
@RequiredArgsConstructor
public class RerankServiceImpl implements RerankService {

    private final ChatClient chatClient;

    private final RagProperties ragProperties;

    @Override
    public List<RetrievedChunk> rerank(String question, List<RetrievedChunk> candidateChunks) {
        if (candidateChunks == null || candidateChunks.isEmpty()) {
            return List.of();
        }

        if (!Boolean.TRUE.equals(ragProperties.getRerank().getEnabled())) {
            return fallbackByFusion(candidateChunks);
        }

        BeanOutputConverter<RerankResultEnvelope> outputConverter =
                new BeanOutputConverter<>(RerankResultEnvelope.class);

        String candidateText = buildCandidateText(candidateChunks);
        String prompt = """
                你是一个 RAG 精排器。
                请根据用户问题，对每个候选片段打分并决定是否保留。

                规则：
                1. score 范围 0 到 100
                2. keep=true 表示建议保留
                3. 高分片段应能直接支持回答用户问题
                4. 结果中的 index 必须对应候选片段中的 index

                用户问题：
                {question}

                候选片段：
                {candidates}

                请严格按下面格式输出：
                {format}
                """;

        try {
            RerankResultEnvelope envelope = chatClient.prompt()
                    .user(user -> user
                            .text(prompt)
                            .param("question", question)
                            .param("candidates", candidateText)
                            .param("format", outputConverter.getFormat()))
                    .call()
                    .entity(outputConverter);

            Map<Integer, RerankDecision> decisionMap = new HashMap<>();
            if (envelope != null && envelope.getResults() != null) {
                for (RerankDecision decision : envelope.getResults()) {
                    if (decision.getIndex() != null) {
                        decisionMap.put(decision.getIndex(), decision);
                    }
                }
            }

            List<RetrievedChunk> rerankedChunks = new ArrayList<>();
            for (int i = 0; i < candidateChunks.size(); i++) {
                RetrievedChunk chunk = candidateChunks.get(i);
                RerankDecision decision = decisionMap.get(i);

                if (decision == null) {
                    continue;
                }

                if (!Boolean.TRUE.equals(decision.getKeep())) {
                    continue;
                }

                double score = decision.getScore() == null ? 0D : decision.getScore();
                if (score < ragProperties.getRerank().getMinScore()) {
                    continue;
                }

                chunk.setRerankScore(score);
                chunk.setSelectedForAnswer(false);
                rerankedChunks.add(chunk);
            }

            if (rerankedChunks.isEmpty()) {
                return fallbackByFusion(candidateChunks);
            }

            return rerankedChunks.stream()
                    .sorted(Comparator.comparing(RetrievedChunk::getRerankScore).reversed())
                    .limit(ragProperties.getRerank().getTopK())
                    .peek(chunk -> chunk.setSelectedForAnswer(true))
                    .toList();
        } catch (Exception e) {
            return fallbackByFusion(candidateChunks);
        }
    }

    private List<RetrievedChunk> fallbackByFusion(List<RetrievedChunk> candidateChunks) {
        return candidateChunks.stream()
                .sorted(Comparator.comparing(RetrievedChunk::getFusionScore).reversed())
                .limit(ragProperties.getRerank().getTopK())
                .peek(chunk -> {
                    chunk.setRerankScore(chunk.getFusionScore() == null ? 0D : chunk.getFusionScore());
                    chunk.setSelectedForAnswer(true);
                })
                .toList();
    }

    private String buildCandidateText(List<RetrievedChunk> candidateChunks) {
        StringBuilder builder = new StringBuilder();

        for (int i = 0; i < candidateChunks.size(); i++) {
            RetrievedChunk chunk = candidateChunks.get(i);
            String content = chunk.getDocument().getText();

            if (content.length() > ragProperties.getRerank().getMaxContentLengthPerChunk()) {
                content = content.substring(0, ragProperties.getRerank().getMaxContentLengthPerChunk());
            }

            builder.append("index=").append(i).append("\n");
            builder.append("sourceFileName=")
                    .append(chunk.getDocument().getMetadata().get("sourceFileName"))
                    .append("\n");
            builder.append("retrievalSource=")
                    .append(chunk.getRetrievalSource())
                    .append("\n");
            builder.append("fusionScore=")
                    .append(chunk.getFusionScore())
                    .append("\n");
            builder.append("content=")
                    .append(content)
                    .append("\n\n");
        }

        return builder.toString();
    }
}
```

这段代码的作用：

- 用结构化输出把“精排”从抽象概念真正落成代码
- 为每条候选片段打一个新的相关性分数
- 把第二阶段的候选结果进一步筛成更适合问答的高质量片段

### 12.3 创建 `src/main/java/com/qiange/ragdemo/service/impl/ContextCompressionServiceImpl.java`

这一步采用“两段式压缩”：

1. 先做规则去重
2. 再做 LLM 压缩
3. 最后做总长度裁剪

```java
package com.qiange.ragdemo.service.impl;

import com.qiange.ragdemo.config.RagProperties;
import com.qiange.ragdemo.service.ContextCompressionService;
import com.qiange.ragdemo.service.model.CompressedContextResult;
import com.qiange.ragdemo.service.model.CompressionDecision;
import com.qiange.ragdemo.service.model.CompressionResultEnvelope;
import com.qiange.ragdemo.service.model.RetrievedChunk;
import lombok.RequiredArgsConstructor;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.converter.BeanOutputConverter;
import org.springframework.stereotype.Service;
import org.springframework.util.StringUtils;

import java.util.ArrayList;
import java.util.LinkedHashMap;
import java.util.List;
import java.util.Map;

/**
 * 第三阶段上下文压缩服务实现。
 *
 * 设计目标：
 * 1. 保留真正有用的片段
 * 2. 尽量去掉重复和冗余描述
 * 3. 把最终上下文控制在更可控的长度范围内
 */
@Service
@RequiredArgsConstructor
public class ContextCompressionServiceImpl implements ContextCompressionService {

    private final ChatClient chatClient;

    private final RagProperties ragProperties;

    @Override
    public CompressedContextResult compress(String question, List<RetrievedChunk> rerankedChunks) {
        if (rerankedChunks == null || rerankedChunks.isEmpty()) {
            return CompressedContextResult.builder()
                    .chunks(List.of())
                    .contextLengthBeforeCompression(0)
                    .contextLengthAfterCompression(0)
                    .build();
        }

        int beforeLength = rerankedChunks.stream()
                .map(chunk -> chunk.getDocument().getText())
                .mapToInt(String::length)
                .sum();

        List<RetrievedChunk> deduplicatedChunks = Boolean.TRUE.equals(ragProperties.getCompression().getDeduplicateBeforeLlm())
                ? deduplicate(rerankedChunks)
                : new ArrayList<>(rerankedChunks);

        List<RetrievedChunk> compressedChunks = Boolean.TRUE.equals(ragProperties.getCompression().getEnabled())
                ? llmCompress(question, deduplicatedChunks)
                : truncateWithoutLlm(deduplicatedChunks);

        List<RetrievedChunk> limitedChunks = limitTotalContext(compressedChunks);

        int afterLength = limitedChunks.stream()
                .map(RetrievedChunk::getCompressedContent)
                .filter(StringUtils::hasText)
                .mapToInt(String::length)
                .sum();

        return CompressedContextResult.builder()
                .chunks(limitedChunks)
                .contextLengthBeforeCompression(beforeLength)
                .contextLengthAfterCompression(afterLength)
                .build();
    }

    private List<RetrievedChunk> deduplicate(List<RetrievedChunk> chunks) {
        Map<String, RetrievedChunk> uniqueMap = new LinkedHashMap<>();

        for (RetrievedChunk chunk : chunks) {
            String normalized = normalize(chunk.getDocument().getText());
            if (!uniqueMap.containsKey(normalized)) {
                uniqueMap.put(normalized, chunk);
            }
        }

        return new ArrayList<>(uniqueMap.values());
    }

    private List<RetrievedChunk> llmCompress(String question, List<RetrievedChunk> chunks) {
        BeanOutputConverter<CompressionResultEnvelope> outputConverter =
                new BeanOutputConverter<>(CompressionResultEnvelope.class);

        String chunkText = buildChunkText(chunks);
        String prompt = """
                你是一个 RAG 上下文压缩器。
                你的任务是针对用户问题，从每个片段中保留最相关的信息。

                规则：
                1. 不要扩写，不要编造
                2. 如果片段与问题无关，keep=false
                3. compressedContent 只保留最关键的信息
                4. 每条 compressedContent 尽量简洁
                5. index 必须对应输入片段中的 index

                用户问题：
                {question}

                输入片段：
                {chunks}

                请严格按下面格式输出：
                {format}
                """;

        try {
            CompressionResultEnvelope envelope = chatClient.prompt()
                    .user(user -> user
                            .text(prompt)
                            .param("question", question)
                            .param("chunks", chunkText)
                            .param("format", outputConverter.getFormat()))
                    .call()
                    .entity(outputConverter);

            List<RetrievedChunk> result = new ArrayList<>();
            if (envelope == null || envelope.getResults() == null) {
                return truncateWithoutLlm(chunks);
            }

            Map<Integer, CompressionDecision> decisionMap = new LinkedHashMap<>();
            for (CompressionDecision decision : envelope.getResults()) {
                if (decision.getIndex() != null) {
                    decisionMap.put(decision.getIndex(), decision);
                }
            }

            for (int i = 0; i < chunks.size(); i++) {
                RetrievedChunk chunk = chunks.get(i);
                CompressionDecision decision = decisionMap.get(i);

                if (decision == null || !Boolean.TRUE.equals(decision.getKeep())) {
                    continue;
                }

                String compressedContent = decision.getCompressedContent();
                if (!StringUtils.hasText(compressedContent)
                        && Boolean.TRUE.equals(ragProperties.getCompression().getDropEmptyChunks())) {
                    continue;
                }

                if (StringUtils.hasText(compressedContent)
                        && compressedContent.length() > ragProperties.getCompression().getMaxCharsPerChunk()) {
                    compressedContent = compressedContent.substring(0, ragProperties.getCompression().getMaxCharsPerChunk());
                }

                chunk.setCompressedContent(compressedContent);
                result.add(chunk);
            }

            return result.isEmpty() ? truncateWithoutLlm(chunks) : result;
        } catch (Exception e) {
            return truncateWithoutLlm(chunks);
        }
    }

    private List<RetrievedChunk> truncateWithoutLlm(List<RetrievedChunk> chunks) {
        List<RetrievedChunk> result = new ArrayList<>();

        for (RetrievedChunk chunk : chunks) {
            String text = chunk.getDocument().getText();
            if (text.length() > ragProperties.getCompression().getMaxCharsPerChunk()) {
                text = text.substring(0, ragProperties.getCompression().getMaxCharsPerChunk());
            }
            chunk.setCompressedContent(text);
            result.add(chunk);
        }

        return result;
    }

    private List<RetrievedChunk> limitTotalContext(List<RetrievedChunk> chunks) {
        List<RetrievedChunk> result = new ArrayList<>();
        int total = 0;

        for (RetrievedChunk chunk : chunks) {
            String content = chunk.getCompressedContent();
            if (!StringUtils.hasText(content)) {
                continue;
            }

            int nextLength = total + content.length();
            if (nextLength > ragProperties.getCompression().getMaxTotalContextChars()) {
                break;
            }

            chunk.setSelectedForAnswer(true);
            result.add(chunk);
            total = nextLength;
        }

        return result;
    }

    private String buildChunkText(List<RetrievedChunk> chunks) {
        StringBuilder builder = new StringBuilder();

        for (int i = 0; i < chunks.size(); i++) {
            builder.append("index=").append(i).append("\n");
            builder.append("content=").append(chunks.get(i).getDocument().getText()).append("\n\n");
        }

        return builder.toString();
    }

    private String normalize(String text) {
        return text.replaceAll("\\s+", " ").trim();
    }
}
```

这段代码的作用：

- 先用规则方法减少重复和噪声
- 再用结构化 LLM 压缩抽出真正相关的信息
- 最后把整段上下文裁剪到一个更可控的长度

### 12.4 修改 `src/main/java/com/qiange/ragdemo/service/impl/RagChatServiceImpl.java`

第三阶段问答链路现在要变成：

1. 改写查询
2. 混合检索
3. 精排
4. 压缩
5. 生成答案

```java
package com.qiange.ragdemo.service.impl;

import com.qiange.ragdemo.common.RagConstants;
import com.qiange.ragdemo.dto.AskQuestionRequest;
import com.qiange.ragdemo.dto.AskQuestionResponse;
import com.qiange.ragdemo.dto.ReferenceChunkResponse;
import com.qiange.ragdemo.service.ContextCompressionService;
import com.qiange.ragdemo.service.HybridRetrievalService;
import com.qiange.ragdemo.service.QueryRewriteService;
import com.qiange.ragdemo.service.RagChatService;
import com.qiange.ragdemo.service.RerankService;
import com.qiange.ragdemo.service.model.CompressedContextResult;
import com.qiange.ragdemo.service.model.RetrievedChunk;
import lombok.RequiredArgsConstructor;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.document.Document;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

/**
 * 第三阶段 RAG 问答服务实现。
 */
@Service
@RequiredArgsConstructor
public class RagChatServiceImpl implements RagChatService {

    private final QueryRewriteService queryRewriteService;

    private final HybridRetrievalService hybridRetrievalService;

    private final RerankService rerankService;

    private final ContextCompressionService contextCompressionService;

    private final ChatClient chatClient;

    @Override
    public AskQuestionResponse ask(AskQuestionRequest request) {
        List<String> queries = queryRewriteService.rewriteQueries(request.getQuestion());
        List<RetrievedChunk> retrievedChunks = hybridRetrievalService.retrieve(queries, request.getCategory());
        List<RetrievedChunk> rerankedChunks = rerankService.rerank(request.getQuestion(), retrievedChunks);
        CompressedContextResult compressedContextResult =
                contextCompressionService.compress(request.getQuestion(), rerankedChunks);

        List<RetrievedChunk> finalChunks = compressedContextResult.getChunks();

        if (finalChunks.isEmpty()) {
            return AskQuestionResponse.builder()
                    .answer("当前知识库中没有足够高质量的上下文支持回答该问题，请先补充知识或调整精排与压缩参数。")
                    .rewrittenQueries(queries)
                    .contextLengthBeforeCompression(compressedContextResult.getContextLengthBeforeCompression())
                    .contextLengthAfterCompression(compressedContextResult.getContextLengthAfterCompression())
                    .references(List.of())
                    .build();
        }

        String context = buildContext(finalChunks);

        String systemPrompt = """
                你是一个严格基于知识库上下文回答问题的助手。
                你的回答必须遵守以下规则：
                1. 优先依据提供的知识片段回答
                2. 如果上下文不足，请明确说明“根据当前知识库无法确定”
                3. 不要编造知识库中不存在的事实
                4. 回答应尽量清晰、结构化、简洁
                """;

        String userPrompt = """
                请基于以下高质量上下文回答问题。

                【上下文开始】
                %s
                【上下文结束】

                用户问题：
                %s
                """.formatted(context, request.getQuestion());

        String answer = chatClient.prompt()
                .system(systemPrompt)
                .user(userPrompt)
                .call()
                .content();

        List<ReferenceChunkResponse> references = finalChunks.stream()
                .map(this::toReferenceChunkResponse)
                .toList();

        return AskQuestionResponse.builder()
                .answer(answer)
                .rewrittenQueries(queries)
                .contextLengthBeforeCompression(compressedContextResult.getContextLengthBeforeCompression())
                .contextLengthAfterCompression(compressedContextResult.getContextLengthAfterCompression())
                .references(references)
                .build();
    }

    private String buildContext(List<RetrievedChunk> chunks) {
        StringBuilder builder = new StringBuilder();

        for (int i = 0; i < chunks.size(); i++) {
            RetrievedChunk chunk = chunks.get(i);
            Document document = chunk.getDocument();
            Map<String, Object> metadata = document.getMetadata();

            builder.append("片段").append(i + 1).append("：").append("\n");
            builder.append("来源文件：")
                    .append(metadata.getOrDefault(RagConstants.METADATA_SOURCE_FILE_NAME, "unknown"))
                    .append("\n");
            builder.append("检索来源：").append(chunk.getRetrievalSource()).append("\n");
            builder.append("融合得分：").append(chunk.getFusionScore()).append("\n");
            builder.append("精排得分：").append(chunk.getRerankScore()).append("\n");
            builder.append(chunk.getCompressedContent()).append("\n\n");
        }

        return builder.toString();
    }

    private ReferenceChunkResponse toReferenceChunkResponse(RetrievedChunk chunk) {
        Document document = chunk.getDocument();
        Map<String, Object> metadata = document.getMetadata();

        return ReferenceChunkResponse.builder()
                .sourceFileName(stringValue(metadata.get(RagConstants.METADATA_SOURCE_FILE_NAME)))
                .sourcePath(stringValue(metadata.get(RagConstants.METADATA_SOURCE_PATH)))
                .chunkIndex(intValue(metadata.get(RagConstants.METADATA_CHUNK_INDEX)))
                .content(document.getText())
                .retrievalSource(chunk.getRetrievalSource())
                .fusionScore(chunk.getFusionScore())
                .rerankScore(chunk.getRerankScore())
                .compressedContent(chunk.getCompressedContent())
                .finalSelectionStatus(chunk.isSelectedForAnswer()
                        ? RagConstants.PIPELINE_SELECTED
                        : RagConstants.PIPELINE_DROPPED)
                .build();
    }

    private String stringValue(Object value) {
        return value == null ? "" : String.valueOf(value);
    }

    private Integer intValue(Object value) {
        if (value == null) {
            return null;
        }
        return Integer.parseInt(String.valueOf(value));
    }
}
```

这段代码的作用：

- 把第三阶段完整在线问答增强链路真正串起来
- 让最终答案建立在“精排后、压缩后”的高质量上下文之上

## 十七、第三阶段常见报错与排查

### 1、问题：精排开启后，结果反而变少甚至为空

优先检查：

1. `rag.rerank.min-score` 是否设置过高
2. 模型返回的 `keep` 是否过于保守
3. 候选集 `candidate-top-k` 是否本来就太少

最常见的调法：

1. 先把 `min-score` 从 `55` 调到 `40`
2. 把 `candidate-top-k` 从 `8` 提到 `10`

### 2、问题：精排结果不稳定

原因通常有两个：

1. 模型本身对评分任务存在波动
2. 候选片段写给模型的文本过长、重点不够聚焦

优先尝试：

1. 降低 `max-content-length-per-chunk`
2. 优化精排提示词，让评分标准更明确
3. 在后续生产化时，再换成独立 Reranker 模型

### 3、问题：压缩后上下文太短，信息丢了

检查：

1. `max-chars-per-chunk` 是否设置太小
2. `max-total-context-chars` 是否设置太小
3. 压缩提示词是不是过于激进

教学阶段建议先从下面这组参数起步：

1. `max-chars-per-chunk=260`
2. `max-total-context-chars=1800`

### 4、问题：压缩后上下文几乎没变短

通常有两种可能：

1. 你的候选片段本来就不长
2. 当前压缩提示词太保守，只是在“原样照搬”

这时候你可以：

1. 优化压缩提示词，要求更聚焦
2. 增加规则截断强度
3. 先通过 `/api/pipeline/debug` 观察每条 `compressedContent`

### 5、问题：流水线调试接口里看到 `compressedContent` 为空

可能原因：

1. 模型认为该片段与问题无关，所以 `keep=false`
2. 压缩结构化输出解析失败，走了降级逻辑
3. 开启了 `drop-empty-chunks=true`

排查方式：

1. 临时关闭 `drop-empty-chunks`
2. 查看是否所有片段都被判定无关
3. 检查结构化输出类字段名是否和提示词约定一致

### 6、问题：最终答案仍然很啰嗦

这说明两种可能：

1. 上下文虽然压缩了，但仍然带了太多相近信息
2. 生成答案的最终提示词还不够收敛

优先从这两步开始：

1. 先看 `/api/pipeline/debug` 的 `compressedChunks`
2. 再优化 `RagChatServiceImpl` 里的 systemPrompt

## 十八、第四阶段还可以继续做什么

如果你已经把第三阶段跑通，下一步就已经不是“基础教程补全”了，而是往更接近生产系统的方向走。

你可以继续往下面几个方向升级：

### 升级 1：替换成独立 Reranker 模型

目标：

- 把当前 LLM 精排替换成更稳定、成本更低的专用精排模型

价值：

- 更接近真实生产方案
- 排序一致性通常更好

### 升级 2：把上下文压缩做成分层压缩

例如：

1. 先句级抽取
2. 再段级合并
3. 最后整体长度裁剪

价值：

- 更细粒度地控制信息密度

### 升级 3：加入最终答案评估

第二阶段我们做了召回评估。

到了第三阶段以后，更值得做的是：

1. 比较压缩前后的答案准确率
2. 比较是否更少幻觉
3. 比较是否更少冗余废话

这一步，才是真正把 RAG 从“检索优化”推进到“答案质量优化”。

## 十九、你在第三阶段真正学到了什么

如果你把这份文档完整敲完并跑通，你真正学到的是下面这些工程事实：

1. 召回到相关片段，不代表最终答案一定会更好
2. 多路召回虽然提高了覆盖率，但也天然会把噪声带进来
3. 精排的本质，是在“覆盖率”和“准确率”之间做更好的平衡
4. 上下文压缩的本质，是在“信息量”和“噪声控制”之间做更好的平衡
5. 一个真正可用的 RAG 系统，核心不只是检索，而是“高质量上下文构建”

这也是为什么教程最后会把重心放在：

1. 精排
2. 压缩
3. 裁剪

因为这些步骤，直接决定了模型最终看到的到底是“垃圾上下文”，还是“高密度证据”。
