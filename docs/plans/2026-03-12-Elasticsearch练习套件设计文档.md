# Elasticsearch 练习套件设计文档

**Date:** 2026-03-12

## 目标

提供 3 个完全独立的 Spring Boot + Elasticsearch 练习项目：

1. `es-product-search-demo`：练习商品搜索、筛选、排序和分页
2. `es-article-search-demo`：练习文章多字段搜索、标签过滤和高亮
3. `es-log-search-demo`：练习日志检索、时间范围过滤和倒序分页

目标读者是 Elasticsearch 初学者。文档必须是中文、步骤化、能照着一步一步创建项目、创建文件、复制代码、启动 Docker 并完成接口测试。

## 为什么选择这 3 个场景

这 3 个场景覆盖了 Elasticsearch 在生产环境里最常见的 3 类用途：

- 电商或后台系统里的商品搜索
- CMS、博客、知识库里的文章全文检索
- 微服务系统里的日志查询和问题排查

它们共同对应了 ES 最核心的能力：

- 全文检索
- 精确过滤
- 范围查询
- 排序与分页
- 搜索结果高亮

## 为什么仍然拆成 3 个独立项目

沿用你前面已经确认过的交付方式：

- 每个场景都是单独的 Spring Boot 项目
- 每个项目都有自己独立的 `docker-compose.yml`
- 每个项目都单独启动自己的 Elasticsearch 容器
- 每个项目的端口、索引名、代码包名完全隔离

这样做的好处是：

- 新手不容易把 3 个场景混在一起
- 每个项目都能单独删除、单独重做
- 你可以按“商品搜索 -> 文章搜索 -> 日志检索”的顺序渐进练习

## 为什么文档固定使用 Elasticsearch 8.10.4

这套文档不是追求“最新版本演示”，而是追求“初学者照着做时尽量少踩兼容性坑”。

因此这里固定使用：

- Spring Boot `3.2.5`
- Elasticsearch Docker 镜像 `8.10.4`

这样做的作用：

- 让 Spring Boot 管理的 ES 相关依赖版本和本地 Docker 环境更接近
- 避免因为大版本差异导致配置方式变化
- 让你在第一次练习时把注意力放在 ES 查询能力上，而不是版本兼容问题上

## 为什么示例数据优先使用英文关键词

这 3 份文档默认不安装中文分词插件，比如 IK。

原因很现实：

- 你现在的目标是先学会 ES 的核心查询能力
- 如果一开始引入额外分词插件，环境复杂度会明显上升
- 很多初学者会把时间浪费在插件安装和版本适配上

所以这次练习里，示例数据会优先使用：

- 英文商品名
- 英文文章标签
- 英文日志服务名

如果你后面要练中文搜索，可以在这套代码跑通后，再继续加 IK 分词器。

## 3 个生产风格场景

### 1. 商品搜索

生产意义：

- 商品搜索通常既要查关键词，也要做分类、品牌、价格区间过滤
- 数据量大之后，关系型数据库的 `LIKE` 查询很难同时兼顾性能和体验

练习目标：

- 建商品索引
- 写入演示商品
- 支持关键词搜索
- 支持分类、品牌、价格、上架状态过滤
- 支持排序和分页

### 2. 文章搜索

生产意义：

- 标题、摘要、正文都可能参与搜索
- 搜索结果通常还需要展示命中高亮，方便用户快速定位内容

练习目标：

- 建文章索引
- 写入演示文章
- 支持多字段全文检索
- 支持分类、标签、作者、状态过滤
- 支持标题和正文高亮

### 3. 日志检索

生产意义：

- 日志排查通常不是简单搜一个词，而是“关键字 + 服务名 + 级别 + 时间范围”的组合检索
- 这类场景正是 ES 最常见的落地用途之一

练习目标：

- 建日志索引
- 写入演示日志
- 支持 message 关键字搜索
- 支持 serviceName、level、traceId 过滤
- 支持时间范围过滤
- 默认按时间倒序分页

- `es-log-search-demo`

2. `es-article-search-demo`
