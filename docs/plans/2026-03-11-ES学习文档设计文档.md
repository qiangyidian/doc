# Elasticsearch Learning Docs Design

**Date:** 2026-03-11

## Goal

Create a beginner-friendly documentation set for three independent Spring Boot Elasticsearch practice projects:

- `es-product-search-demo`
- `es-article-search-demo`
- `es-log-search-demo`

The user should be able to follow the Markdown files step by step to create new modules from an empty workspace without relying on any existing project code.

## Audience

The primary reader is a beginner with limited Spring Boot and Elasticsearch experience. The documents must explain not just what to create, but why each file exists and how the files work together.

## Constraints

- Do not rely on previous repository code.
- Deliver documentation in Markdown.
- Include Docker-based middleware startup instructions.
- Provide module directory structure for each project.
- Explain every file that must be created.
- Provide detailed code comments inside Java and YAML snippets.
- Keep the three projects independent instead of using a shared parent module.

## Chosen Approach

Write one overview guide and three project-specific guides.

1. The overview guide explains real production Elasticsearch scenarios, shared prerequisites, Docker usage, learning order, and the differences between the three practice projects.
2. Each project guide walks through the project from zero:
   - project goal
   - directory structure
   - `pom.xml`
   - Docker Compose
   - Spring Boot configuration
   - startup class
   - document model
   - repository
   - DTOs
   - service interface and implementation
   - controller
   - startup and validation steps
   - common troubleshooting

## Scope

### Product Search

Practice full-text search, filter queries, range queries, sorting, pagination, and highlighting for an e-commerce style scenario.

### Article Search

Practice multi-field text search, tag filtering, author filtering, publishing-time sorting, pagination, and highlighting for a content platform scenario.

### Log Search

Practice keyword search, exact filters, time range filters, descending time sorting, and pagination for an observability-style scenario.

## Deliverables

- `docs/2026-03-11-ES学习指南.md`
- `docs/ES商品搜索示例指南.md`
- `docs/ES文章搜索示例指南.md`
- `docs/ES日志搜索示例指南.md`

## Success Criteria

- The reader can follow the documents without reading other code.
- The reader understands why Elasticsearch is used in each scenario.
- The reader can create all files for a new Spring Boot module from the documentation.
- The reader can start Elasticsearch via Docker Compose and verify the application behavior through HTTP requests.
