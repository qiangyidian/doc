# 主流中间件联合项目落地版开发文档

**目标：** 从零创建一个 `Spring Boot 3` 的“商城中台”示例项目，把当前主流后端开发里最常见的中间件真正串起来使用：`MySQL + MyBatis + Redis + RabbitMQ + Kafka + Elasticsearch + Sa-Token + Docker Compose`。你看完之后，不只是知道这些中间件分别是干什么的，还能按文档一步一步搭出一个完整项目骨架。

**架构说明：** 这份文档采用一个统一的“商城中台”业务场景。MySQL 负责最终真实数据，MyBatis 负责 SQL 映射，Redis 负责缓存与分布式锁，Sa-Token 负责登录与权限，RabbitMQ 负责支付后的业务异步消息，Kafka 负责订单事件流，Elasticsearch 负责商品搜索。所有中间件统一通过 Docker Compose 启动，本地就能完成学习和联调。

**技术栈：** Java 17、Spring Boot 3.2.5、MyBatis 3.0.3、MySQL 8、Redis 7、RabbitMQ 3、Kafka 3.7、Elasticsearch 8.10.4、Sa-Token 1.45.0、Docker Compose、Lombok

---

## 一、为什么要做“联合项目落地版”

如果你只是分别学习：

- Redis 商品缓存
- RabbitMQ 下单通知
- Kafka 订单事件
- Elasticsearch 商品搜索

那你会发现一个问题：

**你会每个组件的单点用法，但不会在一个真实项目里把它们串起来。**

真实后端开发里，更常见的是：

1. 用户先登录后台系统
2. 登录后查商品详情，先走 Redis，Redis 没有再查 MySQL
3. 用户下单时，用 Redis 分布式锁保护库存，再用 MySQL 事务落订单
4. 支付成功后，用 RabbitMQ 发业务异步消息
5. 同时把订单事件发到 Kafka，给日志和统计系统消费
6. 商品发布后把搜索数据同步到 Elasticsearch

所以这份文档的重点是：

- 不再零散学中间件
- 而是学“项目里怎么联合使用”

## 二、你在这份文档里会学到什么

这份文档会覆盖下面这些真实场景：

1. `Sa-Token + Redis`
   做登录认证、Token 校验、用户会话

2. `MySQL + MyBatis + Redis`
   做商品详情缓存

3. `Redis + MySQL`

## 四、先启动所有中间件 `docker-compose.yml`

```yaml
version: "3.9"

services:
  mysql:
    # MySQL 负责保存最终真实数据。
    image: mysql:8.0
    container_name: mall-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: mall_demo
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    command:
      # 使用兼容性更好的认证插件，避免本地练习时驱动连接报错。
      - --default-authentication-plugin=mysql_native_password
    volumes:
      # 把初始化脚本挂载进去，第一次启动容器时会自动执行。
      - ./init/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql

  redis:
    # Redis 用来做商品缓存、Token 持久化、分布式锁。
    image: redis:7
    container_name: mall-redis
    ports:
      - "6379:6379"
    command:
      - redis-server
      - --appendonly
      - yes

  rabbitmq:
    # RabbitMQ 用于支付后的业务异步通知。
    image: rabbitmq:3-management
    container_name: mall-rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"

  kafka:
    # Kafka 用于订单事件流。
    image: apache/kafka:3.7.0
    container_name: mall-kafka
    ports:
      - "9092:9092"
    environment:
      KAFKA_NODE_ID: 1
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_LISTENERS: PLAINTEXT://:9092,CONTROLLER://:9093
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@localhost:9093
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

  elasticsearch:
    # Elasticsearch 用于商品搜索。
    image: docker.elastic.co/elasticsearch/elasticsearch:8.10.4
    container_name: mall-es
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    ports:
      - "9200:9200"
```

这段代码的作用：

- 一次性启动整个项目依赖的所有中间件
- 这就是“联合项目”的基础环境

## 五、创建数据库脚本 `init/mysql/init.sql`

```sql
USE mall_demo;

-- 用户表：用于登录认证。
CREATE TABLE IF NOT EXISTS sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    role_code VARCHAR(50) NOT NULL,
    status TINYINT NOT NULL DEFAULT 1,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 商品表：商品主数据最终保存在 MySQL。
CREATE TABLE IF NOT EXISTS t_product (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    stock INT NOT NULL,
    status TINYINT NOT NULL DEFAULT 1,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 订单主表。
CREATE TABLE IF NOT EXISTS t_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(64) NOT NULL UNIQUE,
    user_id BIGINT NOT NULL,
    total_amount DECIMAL(10, 2) NOT NULL,
    order_status VARCHAR(20) NOT NULL,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
);

-- 订单明细表。
CREATE TABLE IF NOT EXISTS t_order_item (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id BIGINT NOT NULL,
    product_id BIGINT NOT NULL,
    quantity INT NOT NULL,
    item_price DECIMAL(10, 2) NOT NULL,
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_order_id (order_id)
);

-- 初始化管理员和普通用户。
```

## 九、创建统一返回对象和全局异常处理器

### 9.1 创建 `common/Result.java`

```java
package com.example.mallmiddleware.common;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Result<T> {

    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data);
    }

    public static <T> Result<T> fail(String message) {
        return new Result<>(500, message, null);
    }
}
```

这段代码的作用：

- 统一接口返回结构
- 后面无论是登录、查商品、下单还是搜索，都会统一返回这个格式

### 9.2 创建 `common/GlobalExceptionHandler.java`

```java
package com.example.mallmiddleware.common;

import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        // 把所有未处理异常统一拦截，避免把原始堆栈直接暴露给前端。
        return Result.fail(e.getMessage());
    }
}
```

这段代码的作用：

- 给项目加一个最基础的全局异常处理器
- 在联合项目里，这种统一异常兜底几乎是标配

```java
package com.example.mallmiddleware.service;

import com.example.mallmiddleware.entity.ProductInfo;
import com.example.mallmiddleware.mapper.ProductMapper;
import com.fasterxml.jackson.databind.ObjectMapper;
import jakarta.annotation.Resource;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.stereotype.Service;

import java.time.Duration;

@Service
public class ProductService {

    @Resource
    private ProductMapper productMapper;

    @Resource
    private StringRedisTemplate stringRedisTemplate;

    @Resource
    private ObjectMapper objectMapper;

    public ProductInfo queryProduct(Long productId) throws Exception {
        String key = "mall:product:" + productId;

        String cachedJson = stringRedisTemplate.opsForValue().get(key);
        if (cachedJson != null && !cachedJson.isBlank()) {
            return objectMapper.readValue(cachedJson, ProductInfo.class);
        }

        ProductInfo product = productMapper.findById(productId);
        if (product == null) {
            throw new RuntimeException("商品不存在");
        }

        stringRedisTemplate.opsForValue().set(
                key,
                objectMapper.writeValueAsString(product),
                Duration.ofMinutes(30)
        );

        return product;
    }
}
```

这段代码的作用：

```java
package com.example.mallmiddleware.service;

import co.elastic.clients.elasticsearch.ElasticsearchClient;
import co.elastic.clients.elasticsearch.core.SearchResponse;
import com.example.mallmiddleware.entity.ProductDoc;
import com.example.mallmiddleware.entity.ProductInfo;
import com.example.mallmiddleware.mapper.ProductMapper;
import jakarta.annotation.Resource;
import org.springframework.stereotype.Service;

import java.io.IOException;

@Service
public class SearchService {

    @Resource
    private ElasticsearchClient elasticsearchClient;

    @Resource
    private ProductMapper productMapper;

    public void syncProduct(Long productId) throws IOException {
        ProductInfo product = productMapper.findById(productId);
        if (product == null) {
            throw new RuntimeException("商品不存在，无法同步到ES");
        }

        ProductDoc productDoc = new ProductDoc();
        productDoc.setId(product.getId());
        productDoc.setProductName(product.getProductName());
        productDoc.setPrice(product.getPrice());
        productDoc.setStatus(product.getStatus());

        elasticsearchClient.index(i -> i
                .index("product_index")
                .id(String.valueOf(productDoc.getId()))
                .document(productDoc)
        );
    }

    public SearchResponse<ProductDoc> search(String keyword) throws IOException {
        return elasticsearchClient.search(s -> s
                        .index("product_index")
                        .query(q -> q
                                .match(m -> m
                                        .field("productName")
                                        .query(keyword)
                                )
                        ),
                ProductDoc.class
        );
    }
}
```

这段代码的作用：

### 15.8 创建 `service/OrderService.java`

```java
package com.example.mallmiddleware.service;

import cn.dev33.satoken.stp.StpUtil;
import com.example.mallmiddleware.entity.OrderInfo;
import com.example.mallmiddleware.entity.OrderItem;
import com.example.mallmiddleware.entity.ProductInfo;
import com.example.mallmiddleware.kafka.OrderEventProducer;
import com.example.mallmiddleware.mapper.OrderItemMapper;
import com.example.mallmiddleware.mapper.OrderMapper;
import com.example.mallmiddleware.mapper.ProductMapper;
import com.example.mallmiddleware.mq.PaymentSuccessProducer;
import jakarta.annotation.Resource;
import org.redisson.api.RLock;
import org.redisson.api.RedissonClient;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.math.BigDecimal;
import java.time.LocalDateTime;

@Service
public class OrderService {

    @Resource
    private ProductMapper productMapper;

    @Resource
    private OrderMapper orderMapper;

    @Resource
    private OrderItemMapper orderItemMapper;

    @Resource
    private RedissonClient redissonClient;

    @Resource
    private PaymentSuccessProducer paymentSuccessProducer;

    @Resource
    private OrderEventProducer orderEventProducer;

    public String createOrder(Long productId, Integer quantity) {
        // 用 Redis 分布式锁保护同一个商品的并发下单。
        RLock lock = redissonClient.getLock("lock:product:" + productId);
        lock.lock();
        try {
            return doCreateOrder(productId, quantity);
        } finally {
            lock.unlock();
        }
    }

    @Transactional(rollbackFor = Exception.class)
    public String doCreateOrder(Long productId, Integer quantity) {
        ProductInfo product = productMapper.findById(productId);
        if (product == null) {
            throw new RuntimeException("商品不存在");
        }

        int affectRows = productMapper.deductStock(productId);
        if (affectRows == 0) {
            throw new RuntimeException("库存不足");
        }

        String orderNo = "ORD" + System.currentTimeMillis();

        OrderInfo orderInfo = new OrderInfo();
        orderInfo.setOrderNo(orderNo);
        orderInfo.setUserId(StpUtil.getLoginIdAsLong());
        orderInfo.setTotalAmount(product.getPrice().multiply(BigDecimal.valueOf(quantity)));
        orderInfo.setOrderStatus("NEW");
        orderInfo.setCreateTime(LocalDateTime.now());
        orderMapper.insert(orderInfo);

        OrderItem orderItem = new OrderItem();
        orderItem.setOrderId(orderInfo.getId());
        orderItem.setProductId(productId);
        orderItem.setQuantity(quantity);
        orderItem.setItemPrice(product.getPrice());
        orderItem.setCreateTime(LocalDateTime.now());
        orderItemMapper.insert(orderItem);

        // 同时发一个订单创建事件到 Kafka。
        orderEventProducer.sendOrderCreated(orderNo);

        return orderNo;
    }

    public void pay(String orderNo) {
        // 这里为了简化教学，不再补一整套支付表和更新 SQL。
        // 直接模拟支付成功后发 RabbitMQ 消息。
        paymentSuccessProducer.send(orderNo);
    }
}
```

这段代码的作用：

- 这是这份联合项目里最关键的业务服务
- 它把 `Sa-Token + Redis 锁 + MySQL 事务 + Kafka + RabbitMQ` 真正串起来了

## 二十、这份落地文档最重要的 7 个理解点

1. `MySQL` 是最终真实数据源，不要把 Redis、ES 当主库
2. `Redis` 更适合做缓存、锁和登录态
3. `Sa-Token` 解决的是认证与权限，不是业务数据存储
4. `RabbitMQ` 更适合支付成功后的业务异步消息
5. `Kafka` 更适合订单事件流和日志流
6. `Elasticsearch` 更适合搜索，不适合替代 MySQL 做事务主库
7. 联合项目里最重要的是“职责边界清晰”，不是把所有组件硬堆在一起

## 二十二、主流面试题速记

### 22.1 为什么 MySQL 和 Redis 要同时存在

推荐回答：

- MySQL 负责主数据和事务一致性
- Redis 负责高性能访问和热点保护

### 22.2 为什么支付成功后要发 RabbitMQ

推荐回答：

- 支付是主流程
- 短信、积分、通知是从流程
- 异步化后可以降低耦合，提高主流程响应速度

### 22.3 为什么订单事件更适合 Kafka

推荐回答：

- Kafka 更适合高吞吐事件流
- 一个订单事件往往会被多个系统消费

### 22.4 为什么搜索要用 Elasticsearch

推荐回答：

- ES 基于倒排索引，全文检索能力远强于数据库模糊查询

### 22.5 为什么登录态要放 Redis

推荐回答：

- 分布式部署时，多实例要共享登录态
- Redis 是常见的 Session 和 Token 存储位置

如果你只是想知道“中间件分别怎么用”，看单独专题就够了。  
但如果你想真正理解后端项目里的中间件协作方式，这份文档更重要。

真实项目更像这样：

- 再发异步消息
- 再做事件流
- 再做搜索

这份联合项目落地版，核心就是把这整条链路真正串给你看。
