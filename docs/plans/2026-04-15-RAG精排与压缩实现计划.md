# RAG 精排与压缩实现计划

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在当前本地 `Ollama + PGVector` RAG 项目中落地第三阶段的精排、上下文压缩和流水线调试能力。

**Architecture:** 保持第二阶段的检索链路不变，在 `HybridRetrievalService -> RagChatService` 之间插入 `RerankService` 和 `ContextCompressionService` 两个在线问答阶段。类结构和响应形状遵循第三阶段文档，仅在配置层和异常降级路径上适配本地小模型。

**Tech Stack:** Java 17, Spring Boot 3.5, Spring AI 1.1.4, ChatClient, Ollama, PGVector, MyBatis, JUnit 5, Mockito
